Welcome to BotCMS Platform

BotCMS is an open-source, next-generation CMS Platform designed to replace WordPress. It uses Laravel's structured MVC model, an active secure **Hooks Manager**, a dynamic Visual Theme engine (instantly swapping between Tailwind and Bootstrap layouts), and a directory-based Plugin controller. No Composer knowledge or command line required to install plugins or visual components.

1-Click Setup & Installation

Getting BotCMS up and running is simplified via a unified installation wizard. Follow these steps:

STEP 01

Prerequisites

Ensure you have **PHP 8.4+** (Homebrew/apt) and **Composer** installed globally on your hosting server.

STEP 02

Run Installer

Execute `./install.sh` in the project root directory. This starts the interactive installer console.

STEP 03

Choose Database

Select **SQLite** (creates file automatically), **MySQL**, or **PostgreSQL** in the prompts. The script seeds default values.

Developing in BotCMS

Rather than building plugins that write raw SQL or interact with dynamic tables directly, BotCMS organizes business code into separate namespaces:

Modules/

Contains core platform code like **Auth** (login logic) and **Dashboard** (admin screens). Read by `ModuleServiceProvider` at boot.

Plugins/

ZIP-uploadable extensions containing custom controllers, hooks, and routes (like `/seo-audit`). Activated via Admin dashboard.

Themes/

Frontend views and styles. Can specify their required CSS engine in their metadata file.

Theme Structure & Framework Swapping

BotCMS supports multiple frameworks. A theme consists of a folder inside `Themes/` with a `theme.json` configuration file:

1. Tailwind CSS Theme Configuration 
{
  "name": "Default",
  "version": "1.0.0",
  "framework": "tailwind",
  "description": "Tailwind CSS dark mode theme"
}
2. Bootstrap 5 Theme Configuration 
{
  "name": "BootstrapDemo",
  "version": "1.0.0",
  "framework": "bootstrap",
  "description": "Bootstrap 5 light/dark theme"
}
Theme Uploading: To upload a theme, place the theme folder containing `theme.json` and its `resources/views/index.blade.php` structure directly inside the root `Themes/` folder. It will instantly appear as an option in your Admin settings!

How to Edit Basic Items in Theme Views

Views are standard Blade templates. Open the active theme index view located at: Themes/{ActiveTheme}/resources/views/index.blade.php.

Inside the HTML, you can utilize the secure hooks engine directly:

A. Inject dynamic meta tags or custom titles: 
<title>{{ apply_filters('botcms_homepage_title', 'My Default Title') }}</title>
B. Inject custom content or dynamic headers: 
<h1>{!! apply_filters('botcms_homepage_welcome_text', 'Welcome!') !!}</h1>
C. Trigger custom scripts or scripts blocks (Actions): 
@php do_action('botcms_homepage_content_footer'); @endphp

Uploading & Creating Plugins

Plugins are self-contained logical components placed in the root `Plugins/` folder.

1. To build a complex MVC extension (such as an E-commerce store), create a dedicated subfolder (e.g. Plugins/BotCommerce/) containing Models, Controllers, custom Migrations, Routes, and Service Providers.

2. Place a plugin.json descriptor file: 

{
  "name": "BotCommerce",
  "version": "1.0.0",
  "description": "Foundational Product Manager and Shop Catalog plugin for E-Commerce features.",
  "enabled": true
}

3. Create your Service Provider class to register Custom Post Types, dynamic shop routes, submenus tree structure, and hook into core actions/filters: 

namespace Plugins\BotCommerce;
 
use Illuminate\Support\ServiceProvider;
use App\Core\Facades\PostType;
use App\Core\Facades\AdminMenu;
use App\Core\Hooks\Facades\Hook;
 
class BotCommerceServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        // Register Custom Post Type
        PostType::register('product', [
            'label' => 'Products',
            'singular_label' => 'Product',
            'icon' => 'briefcase',
            'supports' => ['title', 'editor']
        ]);
 
        // Register dynamic Admin Menus and Submenus
        AdminMenu::register('botcommerce', [
            'label' => 'BotCommerce',
            'icon' => 'shopping-cart',
            'order' => 30
        ]);
 
        AdminMenu::registerSubmenu('botcommerce', 'products', [
            'label' => 'Products',
            'route' => 'admin.cpt',
            'route_params' => ['type' => 'product'],
            'order' => 1
        ]);
 
        AdminMenu::registerSubmenu('botcommerce', 'gateways', [
            'label' => 'Gateways',
            'route' => 'admin.botcommerce.gateways',
            'order' => 3
        ]);
 
        // Inject custom inputs into CPT edit form using hooks
        Hook::addAction('botcms_cpt_edit_fields_product', function ($post) {
            $controller = app(Controllers\AdminProductController::class);
            echo $controller->renderFields($post);
        });
 
        // Listen to save/update triggers (accepting 2 arguments)
        Hook::addAction('botcms_cpt_saved_product', function ($post, $request) {
            $controller = app(Controllers\AdminProductController::class);
            $controller->saveProduct($post, $request);
        }, 10, 2);
    }
}

Post & Page Creation Structure

Unlike WordPress's heavy EAV database schema, BotCMS organizes pages and posts into a single, clean `posts` table containing:

Field Type Index / Attributes Description
site_id Foreign ID INDEXED Associates the post with a specific virtual site (Multisite scope).
type String INDEXED (post, page, block) Differentiates standard blog posts from static landing pages.
slug String INDEXED (Unique per site+type) Clean URL parameter identifier (e.g. `about-us`).
status String INDEXED (draft, published) Draft, pending review, published, or archived state.

Page Management & Code-First Templates

BotCMS supports a hybrid structure for content creation and page template editing, giving developers and AIs the ultimate flexibility to manage pages through code:

1. Creating Database-Driven Pages (WordPress Style)

Non-technical administrators can manage pages via the admin dashboard:

  1. Log in and navigate to Pages in the sidebar.
  2. Click Create New Page and enter the Title, Slug (e.g. services), and Content.
  3. Select Published and save. The page will instantly resolve at http://127.0.0.1:8000/services.

2. Overriding Database Content with Code (Shopify Theme Style)

If a coding AI or developer wants to build a highly customized layout with animations, custom CSS, or Blade components:

  1. Identify the page slug (e.g. services).
  2. Inside your active theme directory (Themes/{activeTheme}/resources/views/), create a template file named:
    • page-services.blade.php
  3. Put your custom HTML, Tailwind/Bootstrap classes, and Blade logic inside this file.
  4. Resolution Priority: When a user accesses /services, BotCMS checks if the theme template file exists. If it does, it renders the theme template file directly, ignoring the database content! This allows developers to code templates directly in the repo while keeping database content as fallback.

3. Custom Post Types & JSON Metadata (No Bloat CPTs)

Unlike WordPress's slow EAV metadata queries, BotCMS supports registering Custom Post Types (CPTs) with native database JSON column caching:

  1. Register the CPT in AppServiceProvider.php (or inside any Plugin Provider): 
    PostType::register('portfolio', [
    'label' => 'Portfolio',
    'singular_label' => 'Project',
    'icon' => 'briefcase',
    'supports' => ['title', 'editor', 'thumbnail'],
    'fields' => [
        'client_name' => ['type' => 'text', 'label' => 'Client Name'],
        'project_date' => ['type' => 'date', 'label' => 'Completion Date'],
    ]
]);
  2. Instant UI Auto-Generation: A new sidebar menu item is created in the Admin Dashboard automatically. Forms, input fields, and lists are generated dynamically based on the schema and saved inside the native database metadata JSON block.
  3. Template Overrides Signature: Create single-portfolio.blade.php and access custom JSON properties in the Blade template using $post->meta['client_name'].

WordPress vs. BotCMS Migration Guide

To migrate from WordPress to BotCMS, you do not need complex migration plugins. Since BotCMS uses clean Eloquent relational mapping, you can write a simple seeder that reads your WordPress backup XML/JSON and maps posts directly into the `posts` table.
